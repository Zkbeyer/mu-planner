# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev        # Start dev server (localhost:3000)
npm run build      # Runs prebuild (split-courses.js) then Next.js static export → out/
npm run lint       # ESLint via next lint
```

No test suite exists. The `prebuild` step (`node scripts/split-courses.js`) must run before the build — it reads `public/data/courses.json` and `grade_dist.json` and emits `public/data/by-subject/{SUBJECT}.json` + `public/data/search-index.json`. If you modify courses or grade data, re-run `node scripts/split-courses.js` manually before `npm run build`.

## Architecture

### Static export
`next.config.js` sets `output: 'export'` — this is a fully static site with no server runtime. All data is baked in at build time via `getStaticProps`/`getStaticPaths`. The site is deployed to GitHub Pages from the `out/` directory.

### Data pipeline
All source data lives in `public/data/`:
- `courses.json` — full course catalog (~10k courses)
- `grade_dist.json` — per-section GPA records keyed by `courseId`, `term`, `professor`
- `professors.json` — `{ [professorName]: Professor }` object from RMP scrape
- `subjects.json`, `degrees.json`, `college_requirements.json`

**Build-time split** (`scripts/split-courses.js`): Generates `public/data/by-subject/{SUBJECT}.json` (each file contains `{ courses, gpaMap, professorCountMap }`) and `public/data/search-index.json`. The home page lazy-loads these per-subject files client-side when a department is selected.

**Server reads** (`src/lib/data-server.ts`): Used only in `getStaticProps`/`getStaticPaths` — reads the JSON files directly from disk via `fs`. Never imported in client code. Module-level caches (`let _courses = null`) are used in page files to avoid re-reading large files across many static paths.

**Client reads** (`src/lib/data.ts`): Imports JSON directly as ES modules (bundled at build time). Used only where the data is small enough to include in the bundle.

### Pages
- `/` (`src/pages/index.tsx`) — Two-panel layout: left sidebar (department list) + right course grid. Session state is persisted to `sessionStorage` so back-navigation restores scroll position and filters. Global search uses Fuse.js against a lazily loaded `search-index.json`.
- `/course/[id]` — Course detail: instructor table with expandable grade breakdown rows, GPA trend chart, student reviews filtered to this course number.
- `/professor/[slug]` — Professor detail: RMP stats, grade bars per course taught, student reviews. Slug is generated by `professorSlug()` in `src/lib/utils.ts`.

### Styling
All styling uses inline `style` props with CSS custom properties (defined in `globals.css`) plus a small set of Tailwind utility classes. The design system classes (`.card`, `.chip`, `.btn`, `.inp`, `.sec-strip`, `.label`, `.badge`, `.tbl-header`, `.tbl-row`, `.hero-stripe`) are defined in `globals.css` under `@layer components`. Fonts: Syne (headings), Space Grotesk (body), DM Mono (numbers/code).

### Scrapers (`scraper/`)
Python scripts that populate `public/data/`:
- `run_all.py` — orchestrates all four scrapers in order; run via the `scrape.yml` GitHub Action (3× per year, or manual trigger)
- `run_rmp_only.py` — only updates `professors.json`; run via `scrape-rmp.yml` (monthly)
- `scrape_rmp.py` — uses the unofficial RMP GraphQL API; matches professor names from `grade_dist.json` using fuzzy matching (`difflib.get_close_matches`)

### GitHub Actions
- `deploy.yml` — triggers on push to `main`; runs `npm ci && npm run build`, deploys `out/` to GitHub Pages
- `scrape.yml` — full data refresh (grades + catalog + degrees + RMP); commits changes and triggers deploy
- `scrape-rmp.yml` — RMP-only refresh; commits and triggers deploy if data changed
